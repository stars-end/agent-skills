# Spec Template

Use this when drafting a comprehensive implementation plan.

```markdown
# <Title>

## Summary
- One paragraph on the desired outcome.

## Problem
- What is broken, missing, or too expensive today?

## Goals
- Goal 1
- Goal 2

## Non-Goals
- Non-goal 1
- Non-goal 2

## Active Contract
- The exact runtime / operational / product contract that should be true after completion.

## Architecture / Design
- Components, interfaces, and key decisions.

## Execution Phases
1. Phase 1
2. Phase 2
3. Phase 3

## Beads Structure
- Epic: `bd-...`
- Child tasks:
  - `bd-....1`
  - `bd-....2`
  - `bd-....3`

## Validation
- Build/test checks
- Runtime checks
- Human review gates

## Risks / Rollback
- Main risks
- Rollback or containment strategy

## Recommended First Task
- Which child task should start first, and why
```
