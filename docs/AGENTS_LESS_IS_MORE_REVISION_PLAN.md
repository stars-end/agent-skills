Goal: optimize always-loaded context for reliable agent routing by keeping a dense, structured skill index in AGENTS while removing redundant narrative bloat.

## Evaluation Criteria (from prompt)
- Keep WHAT / WHY / HOW; remove everything else from always-loaded baseline.
- Prefer progressive disclosure and pointer-first references.
- Exclude codebase overviews, style guidance prose, and task-specific one-offs.

- `dist/dx-global-constraints.md`: 196 lines

2. Skill catalog is embedded inline in baseline context:

3. Skill corpus is large, and naive \"discover as needed\" is unreliable:

- Therefore, AGENTS must include an explicit skill index (2-3 lines per skill) rather than assuming skill discovery will happen.

4. Baseline currently embeds example markdown with heading markers (`##`) inside code fences, which can pollute section parsing and adds noise.

## Proposed Target State
### Keep in AGENTS baseline (WHAT / WHY / HOW)
- WHAT:
  - canonical repos, canonical Beads contract, core orchestration surfaces.
- WHY:
  - Nakomi constraints + founder cognitive-load policy.
- HOW:
  - canonical command paths for worktree, dispatch, verify, secrets/env sources.
  - non-obvious tool mentions that influence behavior (`uv`, `pnpm`, `mise`, `railway`, `op`, `bd`, `dx-worktree`, `dx-runner`).
  - dense skill index entries that tell agents when to use each skill and where it lives.

### Remove from AGENTS baseline
- Long trigger prose and repetitive narrative per skill.
- Repeated narrative blocks that duplicate `SKILL.md` content.
- Directory/style commentary that is better enforced by tooling.
- situational task instructions that are not globally applicable.

### Replace with progressive disclosure
- Keep a compact but complete skill index in AGENTS (2-3 lines per skill):
  - `when`: short trigger phrase
  - `action`: what the skill does
  - `path`: exact `SKILL.md` location
- Load deep instructions from the referenced `SKILL.md` only when needed.

- Critical invariants retained:
  - worktree-only rule
  - no auto-merge
  - PR metadata requirements
  - secrets/env source contract
