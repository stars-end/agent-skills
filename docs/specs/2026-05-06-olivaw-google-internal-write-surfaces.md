# Olivaw Google Internal-Write Surfaces

Feature-Key: `bd-wf9zw`
Date: 2026-05-06
Owner scope: Olivaw/Hermes Google Workspace lane only

## Decision

Olivaw may perform bounded internal Google Workspace writes for the Star's End
business account `fengning@stars-end.ai`.

This is an `ALL_IN_NOW` implementation for the internal-write surface only.
External-effect actions remain out of scope and are tracked separately in the
blocked follow-up epic.

## Approved Surfaces

The approved execution substrate is:

```bash
scripts/olivaw-gog-safe.sh <gog-subcommand> [args...]
```

State is generated by:

```bash
scripts/olivaw-google-ops-bootstrap.sh
```

The default state file is:

```bash
~/.hermes/profiles/olivaw/google-ops-state.env
```

Approved writes:

- Drive uploads only into approved Olivaw Ops folders.
- Docs create only in approved Olivaw folders.
- Docs write only to Docs already inside approved Olivaw folders.
- Sheets append/update only against the Olivaw Ops Tracker and approved tabs.
- Gmail label create only under `Olivaw/*`.
- Gmail draft create/update only when the subject starts with `[Olivaw`.
- Calendar create/update only on the approved Olivaw calendar, with `[Olivaw ...]`
  event summaries, no guests, no Meet link, and `send-updates=none`.

Approved bootstrap artifacts:

- Drive root: `Olivaw Ops`
- Drive folders:
  - `00 Inbox Drop`
  - `01 Drafts For Review`
  - `02 Approved Working Files`
  - `03 Finance Admin`
  - `04 Healthcare Admin`
  - `05 Reservations`
  - `90 Archive`
  - `99 Audit Logs`
- Sheet: `Olivaw Ops Tracker`
- Sheet tabs:
  - `Intake`
  - `Approvals`
  - `Artifacts`
  - `Reservations`
  - `Finance Admin`
  - `Healthcare Admin`

- Gmail labels:
  - `Olivaw/Inbox`

## Prohibited Actions

The wrapper must block these before API execution:

- Gmail send, forward, autoreply, trash, archive, settings/admin, draft send, and
  draft delete.
- Calendar guests/attendees, Meet link creation, invite/update email, delete,
  move, RSVP, subscribe, calendar creation, and ACL mutation.
- Drive share, unshare, delete, move, rename, copy, replace upload, and runtime
  folder creation.
- Docs copy, delete, clear, format, insert, update, edit, find-replace, tab
  mutation, and sed-style mutation.
- Sheets clear, tab delete, copy, insert, merge, unmerge, and find-replace.
- Any personal Gmail account access from the Olivaw profile.

## Human Approval Gates

The internal-write surface does not approve external commitments.

HITL remains required before:

- sending email,
- inviting guests,
- sharing files externally,
- deleting artifacts,
- submitting forms,
- making reservations,
- calling restaurants,
- submitting healthcare claims,
- initiating Mercury/banking actions,
- or processing live finance/healthcare payloads outside an approved pilot sink.

## Test Plan

Static checks:

```bash
bash -n scripts/olivaw-gog-safe.sh \
  scripts/olivaw-google-ops-bootstrap.sh \
  scripts/olivaw-google-ops-canary.sh
```

Auth and setup:

```bash
scripts/olivaw-gog-safe.sh auth doctor --check --json
scripts/olivaw-google-ops-bootstrap.sh
```

Runtime canary:

```bash
scripts/olivaw-google-ops-canary.sh
```

If the Gmail OAuth token lacks label/draft write scopes, bootstrap may be run
with `OLIVAW_SKIP_GMAIL_SETUP=1` to finish Drive, Docs, Sheets, Calendar, and
state-file setup while keeping Gmail write work blocked until re-auth.

Manual verification:

- Confirm the `Olivaw Ops` folder tree exists in Google Drive.
- Confirm `Olivaw Ops Tracker` tabs and headers exist.
- Confirm synthetic Docs, Drive upload, Calendar hold, and Gmail draft canaries
  are synthetic only and contain no sensitive payloads.
- Confirm blocked-action probes return policy failures without external effects.
